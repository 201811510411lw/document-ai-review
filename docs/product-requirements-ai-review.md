# 私有化智能审核平台产品需求

本文档归档 `document-ai-review` 的目标产品范围。它描述产品定位、业务场景、输入输出和约束，不承载实现细节。

---

## 1. 产品定位

`document-ai-review` 是面向企业内部文档审核的私有化智能审核平台。平台把证照、批次报告、检验报告、合同等非结构化材料转为可校验、可追溯、可复核的结构化审核结果。

平台不是单一 OCR 工具，也不是单一 `food_license` 审核服务。OCR、LLM、PDF、图片解析、OA 回写和 IM 通知都是可替换 Adapter。当前主线的产品能力边界依赖三层结构：

- Agent Skill 描述层：沉淀审核能力边界；
- runtime use_case：承接业务场景入口；
- runtime capability：承接可复用执行能力。

产品目标：

- 支持企业内部私有化部署；
- 保证敏感文档和抽取数据不出内网；
- 将人工审核经验沉淀为可测试的确定性规则和可复核结果；
- 通过统一 ReviewService 和 `use_case_registry` 暴露审核入口；
- 为不同审核场景保留独立 use_case 演进空间，并通过 capability 复用执行能力。

## 2. 业务场景

### 2.1 QC 证照及批次报告审核

审核对象：

- 供应商证照；
- 营业执照；
- 食品经营许可证；
- 食品生产许可证；
- 商品批次报告；
- 第三方检验报告。

输入：

- OCR 文本；
- 图片或 PDF 文件；
- 文件路径或对象存储引用；
- 供应商名称、统一社会信用代码、经营地址等主数据；
- 商品、批次、报告类型等业务上下文。

输出：

- 文档类型识别结果；
- 关键字段抽取结果；
- 字段规范化结果；
- 规则校验结果；
- 风险等级和审核摘要；
- 是否需要人工复核及复核原因。

### 2.2 营业执照与烟草证一致性校验

审核对象：

- 营业执照 OCR 结果；
- 烟草专卖零售许可证 OCR 结果；
- OA 或业务系统中的主体信息。

输入：

- 营业执照 OCR 文本或文件；
- 烟草专卖零售许可证 OCR 文本或文件；
- 企业名称、经营场所、负责人、有效期等业务字段；
- OA 流程单号或外部业务引用。

输出：

- 双证字段抽取结果；
- 名称、经营场所、负责人、有效期字段比对结果；
- 风险等级和不一致说明；
- 是否需要人工复核；
- OA 回写 payload 摘要；
- 飞书或企微告警摘要。

### 2.3 法务合同内容审核

审核对象：

- 通用合同；
- 房屋租赁合同；
- 常温食品供货合同；
- 合同附件和补充协议。

输入：

- 合同 OCR 文本；
- PDF、Word 转换文本或文件引用；
- 合同类型、相对方、金额、期限、业务部门等上下文；
- OA 合同流程信息。

输出：

- 合同类型识别结果；
- 关键条款抽取结果；
- 风险条款识别结果；
- 修改建议摘要；
- 风险等级；
- 是否需要人工复核或法务复核；
- OA 合同流程回写 payload 摘要。

## 3. 非功能约束

- 私有化部署：平台默认面向企业内网部署，不把核心审核链路绑定到公有云服务；
- 数据不出内网：证照、报告、合同文本、抽取字段和审核结果默认保留在内部环境；
- 不依赖公有云大模型 API：OCR 和 LLM 能力必须通过 Adapter 接入，可替换为本地模型、内网服务或测试 Stub；
- 可追溯：审核任务、规则结果、人工复核状态和审计事件需要形成结构化记录；
- 可测试：确定性业务规则必须能在本地测试中验证，不依赖真实 OCR、真实 LLM、OA、ERP 或 IM 服务；
- 可扩展：新增审核场景应通过新增 Agent Skill 描述、use_case、workflow、capability、rules 和 Adapter 组合完成，不应让平台层直接调用场景内部节点。

## 4. 当前 V1 实现范围

当前代码已经具备：

- FastAPI 服务骨架；
- ReviewService 审核入口；
- `use_case_registry` 显式注册骨架；
- `food_license` Python use_case facade；
- `food_license` runtime capability 样板；
- `food_license` LangGraph 工作流；
- `ReviewResult` 平台级返回契约；
- `skill_result` 承载 capability 专属 payload 的兼容边界；
- 食品安全证照 OCR 文本输入、本地 PDF、文件输入边界的最小测试闭环。

当前 V1 本轮扩展范围：

- 保留多场景 Agent Skill 描述层；
- 保留多场景 workflow 目录骨架；
- 保留 OCR、LLM、文档、文件、PDF、图片、ERP、OA、IM Adapter Stub；
- 保持 `use_case_registry` 和 ReviewService 的多场景调用骨架；
- 保留 `food_license` 兼容入口；
- 优先收口 use_case、capability 和 Agent Skill 的命名边界。

---

## 5. 当前未实现范围

本阶段不实现：

- 完整 QC 证照及批次报告审核规则；
- 完整营业执照与烟草证一致性校验规则；
- 完整法务合同审核规则；
- 真实 OCR 接入；
- 真实 LLM 接入；
- 真实 ERP、OA、飞书或企微调用；
- 数据库规则配置、规则热加载或租户级规则覆盖；
- 删除 `food_license` 兼容入口；
- 将旧分支上的实验能力整包合并回主线。

当前主线优先做架构、术语和边界收口，不继续扩展 `food_license` 业务能力。
